## 1. 项目背景
随着 AI 编程助手（如 Claude Code）的普及，团队在享受效率提升的同时，面临以下挑战：

+ AI 生成的代码难以追溯和审查；
+ AI 引入的 bug 缺乏系统记录；
+ 代码审查过程与 AI 修改脱节；
+ 团队知识（AI 使用经验、常见问题）无法沉淀。

为解决上述问题，需开发一款 PC 小工具（Python 实现），作为统一的 AI 编程接口，自动记录 AI 修改的代码、bug 提交和审查过程，最终形成公司内部的知识库。

## 2. 用户角色
+ **开发者**：日常使用 AI 辅助编程，提交代码和 bug，参与代码审查。
+ **技术负责人 / 架构师**：查看知识库，分析 AI 使用趋势，优化团队规范。
+ **AI 管理员**：配置 AI 模型参数、管理接入凭证。

## 3. 功能需求
### 3.1 统一的 AI 编程接口
+ **多模型支持**：至少支持 Claude Code，预留扩展接口以便后续接入其他 AI 模型（如 DeepSeek、OpenAI）。
+ **交互方式**：
    - 命令行界面（CLI）：`ai-tool ask "编写一个排序函数"`
    - 图形用户界面（GUI）：对话式窗口，实时显示 AI 回复。
+ **对话管理**：支持多轮对话，记录上下文，保证连续需求理解。

### 3.2 代码修改记录
+ **自动捕获变更**：
    - 记录修改前后代码片段、项目、文件路径、修改时间、用户 Prompt。
    - 开发者接受 AI 代码时自动入库。
+ **版本控制集成**：
    - 自动检测 Git/SVN 仓库。
    - 自动创建备份分支：`ai-temp-{timestamp}`。
    - 可选自动提交，信息格式：`[AI] {指令摘要}`。
+ **历史查询**：按项目、时间、关键词筛选查看 AI 修改记录。

### 3.3 Bug 提交记录
+ **结构化记录**：标题、描述、复现步骤、自动关联 AI 修改记录。
+ **错误捕获**：支持捕获代码上下文、错误堆栈。
+ **状态管理**：待处理、修复中、已关闭。
+ **修复关联**：记录修复代码，与原 AI 修改对比复盘。

### 3.4 代码审查记录
+ **审查流程**：
    - 标记 AI 修改需审查，邀请成员参与。
    - 展示代码 diff，支持行评论、评分（通过 / 需修改）。
    - 记录结论，支持自动合并等后续操作。
+ **审查历史**：永久保存意见、讨论、决策，支持搜索与导出。

### 3.5 知识库构建
+ **数据汇聚**：
    - 本地 SQLite 存储所有记录。
    - 支持全文检索（指令、代码、评论等）。
+ **知识沉淀**：
    - 自动生成报表：模型 Bug 率、错误模式、高频指令。
    - 导出为 Markdown/HTML。
+ **智能推荐**：相似历史指令与方案推荐。

### 3.6 与 Claude Code 集成
+ **调用方式**：优先 API，备选剪贴板 / 模拟输入。
+ **配置管理**：API Key、模型参数（温度、token 等）可视化配置，支持多用户共享配置文件。

### 3.7 与 Git/SVN 集成
+ 基础操作：创建临时分支、自动提交、差异对比。
+ 冲突处理：未提交变更时提示手动提交，避免冲突。

## 4. 非功能需求
### 4.1 性能
+ UI 响应 < 100ms，AI 请求异步不阻塞。
+ 常见搜索 < 1 秒。
+ 启动时间 < 2 秒。

### 4.2 安全性
+ 敏感信息使用系统密钥环（keyring）或加密存储。
+ 默认本地存储，不上传云端。

### 4.3 可扩展性
+ 插件式 AI 接口（BaseAI 抽象类）。
+ 数据库预留扩展字段。

### 4.4 易用性
+ CLI 简洁友好，自带帮助。
+ GUI 遵循 Fluent Design，流程清晰。
+ 首次启动配置向导。

### 4.5 跨平台
支持 Windows 10/11、macOS、主流 Linux 发行版。

## 5. UI/UX 需求
### 5.1 技术选型
+ GUI：PySide6 / PyQt5 + PyQt-Fluent-Widgets
+ 异步：Qt Signal + Slot，耗时操作放入工作线程

### 5.2 界面布局
+ 主窗口：左侧导航 + 右侧内容区
+ 导航模块：AI 对话、修改历史、Bug 追踪、代码审查、知识库、设置

#### 各页面结构
+ **AI 对话**：模型选择 + 对话区 + 多行输入框
+ **历史记录**：表格列表 + 详情弹窗（diff、关联 Bug）
+ **Bug 追踪**：状态列表 + 详情（修复方案）
+ **代码审查**：待审查列表 + diff 对比 + 评论区
+ **知识库**：搜索 + 筛选 + 统计图表

### 5.3 交互细节
+ 耗时操作显示加载动画，禁用重复提交。
+ 系统通知 / 状态栏提示。
+ 支持浅色 / 深色主题切换。

## 7. 工作流程示例
1. 开发者输入：实现斐波那契数列函数。
2. 工具异步调用 Claude，返回代码。
3. 点击「应用修改」：创建 Git 临时分支 → 写入文件 → 记录 diff、Prompt、时间。
4. 发现 Bug，新建记录并自动关联 AI 修改。
5. 负责人审查，标记「需修改」。
6. 开发者修正后重新提交，审查通过。
7. 所有记录入库，可搜索「斐波那契」查看历史方案。
